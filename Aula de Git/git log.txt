No Git, "log" refere-se ao histórico de commits em um repositório. O comando git log exibe uma lista de commits realizados, permitindo que você veja informações detalhadas sobre cada commit, como:

Hash (ID) do commit
Autor do commit
Data e hora do commit
Mensagem de commit
Para visualizar o log de commits, você pode usar o comando:

git log

Esse comando exibirá uma lista de commits com detalhes como:

commit 1a2b3c4d5e6f7g8h9i0j
Author: Seu Nome <seuemail@example.com>
Date:   Mon Jan 1 12:34:56 2023 -0300

    Mensagem do commit

commit 0j9i8h7g6f5e4d3c2b1a
Author: Outro Nome <outroemail@example.com>
Date:   Sun Dec 31 11:33:55 2022 -0300

    Outra mensagem de commit

Existem várias opções que você pode usar com git log para personalizar a saída. Algumas das mais comuns incluem:

--oneline: Exibe cada commit em uma linha.
--graph: Exibe um gráfico ASCII do histórico de commits.
--decorate: Adiciona referências (branch, tag) aos commits.
--stat: Mostra um resumo das alterações realizadas em cada commit.
Exemplo de uso:

git log --oneline --graph --decorate --stat

Isso fornecerá uma visão condensada e visual do histórico de commits, incluindo a estrutura de branches e merges.
